5. Example or scenario of speaking from memory


Answer:

Speak from Memory

Uses of Memorized Deliver

Good orators of the past were expected to deliver their speeches from memory without notes or other aids. Today, you may find it to your advantage to memorize your speech or presentation.

Today, memorization takes two basic forms:

The total speech or manuscript is committed to memory.

Standard parts of a message are memorized and woven into each speech.

Advantages

If you memorize your speech you are more likely to perform better than the speaker reading from a manuscript, since you can stay in touch with your audience. When speaking from memory, you do not need to think about what is coming next or how to express an idea as you would in extemporaneous speaking; therefore, you can give more attention to the audience. Today, we generally admire and stress extemporaneous speaking in our society, but many of the great speeches in history were delivered all or partially from memory.

Disadvantages

One of the main problems of delivering a speech from memory is sounding rehearsed, or like you are reading. It is possible to memorize and deliver a speech with variety and emphasis.

Tips for the Speaker

You may use various approaches to memorize and rehearse your speech. Below are some guidelines to follow when delivering a memorized speech:

When writing the speech, write as if you were speaking naturally, directly to an audience.

Develop an outline of the main points and then memorize the outline.

If delivering the same speech to different audiences over time, memorize the individual sections and then weave them together for each occasion.

If you forget a word or two or a small section, just continue speaking.

Overall, if you have the attitude that you are communicating with an audience rather than reciting words, you are likely to succeed.

Event-driven programming is a programming paradigm in which the flow of the program is determined by events such as user actions (clicks, keystrokes), messages from other programs, or changes in the system's state. Here are some examples of event-driven programming:

Graphical User Interface (GUI) Applications: In a GUI application, the program waits for user events such as button clicks or menu selections, and responds accordingly by updating the interface or performing some action. Examples of GUI applications include web browsers, text editors, and media players.

Games: Games are often event-driven, where the game responds to user input, such as pressing a button, moving the mouse, or typing on the keyboard. The game updates the display and changes the game state based on the input received from the user.

Web Development: JavaScript is a popular language for web development, and it is commonly used for event-driven programming. In web development, the browser waits for events such as button clicks, form submissions, or page loads, and responds by updating the page or making requests to the server.

Robotics: Robotics programming often involves event-driven programming, where the robot waits for sensor inputs such as light or sound, and responds by moving or performing some action. For example, a robot might wait for a button press or a voice command, and then move or speak accordingly.

Internet of Things (IoT) Applications: IoT devices often use event-driven programming, where the device waits for events such as a change in temperature, a motion sensor trigger, or a message from the cloud, and responds by sending a message or performing some action.

21. object oriented event driven programming EXPLAIN AND EXAMPLE what is event driven programming memory java explain with example


Object-oriented event-driven programming is a programming paradigm in which software components (objects) are designed to respond to events that occur within a program or system. Events can include user interactions (such as mouse clicks or key presses), system events (such as messages from other programs or devices), or changes to data within the program.

In an event-driven program, the flow of execution is determined by events, rather than being strictly sequential. When an event occurs, the appropriate object or objects are notified, and they respond by executing predefined methods or functions. This makes it possible to create dynamic, interactive programs that can respond to user input in real-time.

One common example of event-driven programming is graphical user interface (GUI) programming. In this context, events can include mouse clicks, window resizing, and other interactions with the user interface. When an event occurs, the appropriate object or objects (such as buttons or text fields) are notified, and they respond by executing methods or functions to update the display or perform other actions.

Memory management in Java for event-driven programming involves the use of a garbage collector, which automatically frees up memory that is no longer being used by the program. This allows developers to focus on writing code to respond to events, rather than having to manage memory manually.

For example, in Java Swing, an event-driven GUI toolkit, a programmer might create a button object and attach an action listener to it. When the user clicks the button, the action listener is notified, and it executes the appropriate method or function. Here's some sample code to illustrate this:

import javax.swing.*;

import java.awt.*;

import java.awt.event.*;

public class MyButtonApp extends JFrame implements ActionListener {

private JButton myButton;

public MyButtonApp() {

super("My Button App");

myButton = new JButton("Click me!");

myButton.addActionListener(this);

getContentPane().add(myButton, BorderLayout.CENTER);

pack();

setVisible(true);

}

public void actionPerformed(ActionEvent e) {

JOptionPane.showMessageDialog(this, "You clicked the button!");

}

public static void main(String[] args) {

MyButtonApp app = new MyButtonApp();

}

}

In this code, we define a new class called MyButtonApp, which extends the JFrame class and implements the ActionListener interface. We create a new JButton object called myButton, and attach an ActionListener object (which in this case is the MyButtonApp object itself) to it using the addActionListener method. When the button is clicked, the actionPerformed method is called, which displays a dialog box using the JOptionPane class. The main method creates a new instance of the MyButtonApp class, which creates the GUI and displays it on the screen.
